## What I Learned

- `let` declarations aren't accessible from `window` scope — need `var` for E2E test access
- Variable declaration order matters with `let` — calling initBgStars before declaration causes ReferenceError
- Canvas game loop with requestAnimationFrame at 60fps is smooth and performant
- E2E tests for games need to be deterministic — teleporting player or force-spawning is more reliable than playing through
- Lighter gravity (0.22 vs 0.35) makes the game much more enjoyable and controllable

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| Vanilla HTML/CSS/JS | No build tools needed, simplest approach |
| Canvas over DOM | Better performance for 60fps game |
| `var` for game state | Enables Playwright E2E tests via `page.evaluate()` |
| SemVer versioning | User requested version display with semver bumps per feature |
| Gravity 0.22 | User requested lighter fall physics |
| Playwright for E2E | Reliable browser automation, integrates well with the project |
