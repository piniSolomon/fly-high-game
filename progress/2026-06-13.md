## What I Learned

- `var` scope needed for Playwright E2E test access
- Web Audio API procedural sounds avoid external file dependencies
- `effectiveSpeed` variable cleanly separates base speed from power-up modifiers
- Pause requires checking flag at top of update — simple but must not forget
- Power-up hexagonal shapes with glow stand out well against star field background
- Magnet pull with distance-based falloff feels natural

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| 3 power-up types | Shield (defensive), Magnet (collection), Slow-mo (control) cover different play styles |
| 5-second power-up duration | Long enough to feel useful, short enough to not trivialize the game |
| Escape for pause | Standard game convention |
| effectiveSpeed pattern | Clean way to apply slow-mo without touching every scroll line individually |
