## What I Learned

- Canvas-based game loop with requestAnimationFrame works well for this game type
- Particle effects add significant visual polish for minimal code
- The game is playable but feels static — needs horizontal scrolling to feel like a flying game
- Touch controls work with passive: false on touchstart/touchend

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| Canvas over DOM elements | Better performance for 60fps game with many moving objects |
| Circle-based collision | Simple and effective for stars + player |
| 8 stars with respawn | Keeps the field populated without overwhelming the player |
| Death on boundary hit | Simple rule that creates tension without complex obstacle logic |
